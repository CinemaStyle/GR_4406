# **Шпаргалка по командам**

## Базовые команды (Basic git)

### git init <directory.>
>Создайте пустой репозиторий git в указанном каталоге. Запустите без аргументов, чтобы инициализировать текущий каталог как репозиторий git.

### git clone <repo.>
>Клонирует репозиторий, расположенный в <repo.>, на локальный компьютер. Исходный репозиторий может быть расположен в локальной файловой системе или на удаленном компьютере через HTTP или SSH.

### git config user.name <name.>
>Определите имя автора, которое будет использоваться для всех коммитов в текущем репозитории. Разработчики обычно используют флаг --global для установки параметров конфигурации для текущего пользователя.

### git add <directory.>
>Подготовьте все изменения в <directory.> для следующего коммита. Замените <каталог.> на <файл.>, чтобы изменить конкретный файл

### git commit -m <message.>
>Зафиксируйте промежуточный результат, но вместо запуска текстового редактора используйте <message.> в качестве комментария к коммиту.

### git status 
>Выведите список того, какие файлы помещены в архив, не помещены в каталог и не отслеживаются.

### git log
> Отображение всей истории коммитов в формате по умолчанию. 

### git diff
>Показать неустановленные изменения между вашим индексом и рабочим каталогом

## Редактирование (rewriting git history)

### git commit --amend
>Замените последний коммит этим коммитом. Используйте без подготовки, чтобы отредактировать сообщение последнего коммита.

### git rebase <base.>
>Перебазируйте текущую ветку на <base.>. <base.> может быть идентификатором фиксации, именем ветки, тегом или относительной ссылкой на HEAD.

### git reflog
>Показать журнал изменений в HEAD локального репозитория. Добавьте флаг --relative-date для отображения информации о дате или --all для отображения всех ссылок.

## Отмена изменений (Undoing changes)

### git revert <commit.>
>Создайте новый коммит, который отменяет все изменения, сделанные в <commit.>, затем примените его к текущей ветке.

### git reset <file.>
>Удалите <file.> из промежуточной области, но оставьте рабочий каталог без изменений. Это удаляет файл без перезаписи каких-либо изменений

### git clean -n
>Показывает, какие файлы будут удалены из рабочего каталога. Используйте флаг -f вместо флага -n для выполнения очистки.

## Ветки (Branches git)

### git branch 
>Перечислите все ветки в вашем репозитории. Добавьте аргумент, чтобы создать новую ветвь с именем.

### git checkout -b <branch.>
>Создайте и извлеките новую ветку с именем <branch>. Уберите флаг -b, чтобы проверить существующую ветку.

### git merge <branch.>
>Объединить <branch.> с текущей ветвью.

## Удаленные репозитории (Remote repositories)

### git remote add <name.> <url.>
>Создайте новое подключение к удаленному репозиторию. После добавления вы можете использовать <name.> в качестве ярлыка для <url.> в других командах.

### git fetch <remote.> <branch.>
>Выбирает конкретную <branch.> из репозитория. Оставьте <branch.>, чтобы получить все удаленные ссылки.

### git pull <remote.>
>Получить указанную удаленную копию текущей ветки и немедленно объединить ее с локальной копией.

### git push <remote.> <branch.>
>Отправьте ветку на <remote.> вместе с необходимыми коммитами и объектами. Создает именованную ветку в удаленном репозитории, если она не существует.

## Конфигурация (Config git)

### git config --global user.name <name.>
> Определите имя автора, которое будет использоваться для всех коммитов текущим пользователем.

### git config --global user.email <email.>
>Определите адрес электронной почты автора, который будет использоваться для всех коммитов текущим пользователем.

### git config --globalalias. <alias-name.> <git-command.>
>Создать ярлык для команды git. Например. alias.glog «log --graph --oneline» установит «git glog» эквивалентно «git log --graph --oneline».

### git config --system core.editor <editor.>
>Установите текстовый редактор, используемый командами для всех пользователей на машине. <editor.> аргумент должен быть командой, которая запускает нужный редактор (например, vi)

### git config --global --edit
>Откройте файл глобальной конфигурации в текстовом редакторе для редактирования вручную.

## Разница (Diff git)

### git diff 
>Показать разницу между рабочим каталогом и последним коммитом.

### git diff --cached
>Показать разницу между поэтапными изменениями и последней фиксацией

## Сброс (Reset git)

### git reset
>Сбросьте промежуточную область, чтобы она соответствовала самой последней фиксации, но оставьте рабочий каталог без изменений.

### git reset --hard
>Сбросить промежуточную область и рабочий каталог, чтобы они соответствовали самой последней фиксации, и перезаписать все изменения в рабочем каталоге.

### git reset <commit.>
>Переместите конец текущей ветки назад к <commit.>, сбросьте промежуточную область, чтобы она соответствовала, но оставьте рабочий каталог в покое.

### git reset --hard <commit.>
>То же, что и предыдущее, но сбрасывает и промежуточную область, и рабочий каталог, чтобы они совпадали. Удаляет незафиксированные изменения и все фиксации после <commit.>.

## Перебазирование (Rebase git)

### git rebase -i <base.>
>Интерактивно перебазировать текущую ветку на <base.>. Запускает редактор для ввода команд того, как каждая фиксация будет перенесена в новую базу.

## Журнал (Log git)

### git log -<limit.>
>Ограничить количество коммитов на <limit.>. Например. «git log -5» ограничит до 5 коммитов.

### git log --oneline
>Сократите каждый коммит до одной строки

### git log --stat
>Укажите, какие файлы были изменены, и относительное количество строк, которые были добавлены или удалены из каждого из них.

### git log -p
>Показать полную разницу каждого коммита.

### git log --author=”<pattern.>”
>Поиск коммитов определенного автора.

### git log --grep=”<pattern.>”
>Поиск коммитов с сообщением коммита, соответствующим <pattern.>.

### git log <since.>..<until.>
>Показать коммиты между <since.> и <until.>. Аргументы могут быть идентификатором фиксации, именем ветки, HEAD или любым другим видом ссылки на версию.

### git log -- <file.> 
>Отображать только те коммиты, которые имеют указанный файл.

## Получение (Pull git)

### git pull --rebase <remote.>
>Получить удаленную копию текущей ветки и перебазировать ее в локальную копию. Использует git rebase вместо слияния для интеграции веток.

## Отправление (Push git)

### git push <remote.> --force
>Принуждает git push, даже если это приводит к слиянию без быстрой перемотки вперед. Не используйте флаг --force, если вы абсолютно не уверены, что знаете, что делаете.

### git push <remote.> --all 
>Нажмите все ваши локальные ветки на указанные удаленные.

### git push <remote.> --tags
>Теги не добавляются автоматически при отправке ветки или использовании флага --all. Флаг --tags отправляет все ваши локальные теги в удаленное репозитории.
